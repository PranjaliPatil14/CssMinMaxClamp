min(), max() and clamp()

min(a1, a2, a3, ...)

selects minimum value from all the arguments
example: min(40%, 600px)

width: 2500px
40%: 40*2500/100 = 1000px
min(600px, 1000px) = 600px

width: 1667px
40%: 40*1667/100 = 666px
min(600px, 666px) = 600px

width: 1330px
40%: 40*1330/100 = 532px
min(600px, 532px) = 532px

max(a1, a2, a3, ...)

selects maximum value from the arguments
example : max(40%, 600px);

width: 2500px
40%: 40*2500/100 = 1000px
max(600px, 1000px) = 1000px

width: 1667px
40%: 40*1667/100 = 666px
max(600px, 666px) = 666px

width: 1330px
40%: 40*1330/100 = 532px -> 600px
max(600px, 532px) = 600px

.
.
.
i.e max will take 600px whenever 40vw is less than 600px.
i.e max is setting min-width value for container.
 Example
  1. Can nest max() and other min() functions as expression values.
  2. Can do mathematical operations inside max() or min() no need to use calc(). example: min(40% - 20px , 200px);

clamp(minValue, preferredValue, maxValue)

  • clamps a value between min value and max value.
  • clamps = max(minValue, min(preferredValue, maxValue))
example: clamp(350px, 50%, 700px)

width: 2550px;
50%: 1275px
clamp(350px, 1275px, 700px) -> 700px

width: 1300px;
50%: 665px
clamp(350px, 665px, 700px) -> 665px

width: 900px;
50%: 450px
clamp(350px, 450px, 700px) -> 450px

width: 600px;
50%: 300px
clamp(350px, 300px, 700px) -> 350px

i.e whenever preferredValue is greater than max value, max value will be considered.
i.e whenever preferredValue is less than min value, min value will be considered.
Example
clamp has its minimum value "win" over its maximum value if the two are in the "wrong order".

That is, clamp(100px, ..., 50px) will resolve to 100px, exceeding its stated "max" value.

If alternate resolution mechanics are desired they can be achieved by combining clamp() with min() or max():

To have MAX win over MIN:

clamp(min(minValue, maxValue), preferredValue, maxValue) or min(maxValue, max(minValue, preferredValue)).
